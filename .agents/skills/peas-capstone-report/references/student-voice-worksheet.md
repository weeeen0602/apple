# 學生口述工作表（Step 1–3 → 3c 對齊條列）

Agent 在 **Step 3c** 將學生回答整理為下列格式。學生確認後，§ 編號對應 `report-template.md` 章節。

## 引導問題（Agent 心內 checklist）

### Step 1 專題介紹

- 專題中文名稱？
- 一句話價值主張？
- 目標使用者？
- 痛點／動機？

### Step 2 左欄↔右欄

- 自訂頁有幾個？各做什麼（白話）？
- 左欄 → 右欄：什麼狀態／資料會跟著問題送出去？
- 右欄 → 左欄／檔案：Agent 或工具寫回哪裡？
- 一條完整 user journey？

### Step 3 成果／創新／技術

- 可 demo 功能列表
- 與模板或他組不同之處
- 技術關鍵字（Streamlit、session_state、JSON、tool call…）— 只列學生實際用到的

---

## 對齊條列輸出模板（Step 3c 貼給學生）

```markdown
## 對齊條列（請確認）

### §1 專題介紹
- 專題名稱：……
- 一句話說明：……
- 主要使用者：……
- 解決的問題：……

### §2 左欄↔右欄互動
- 自訂頁：……（頁名／用途）
- 左欄 → Agent：……
- Agent → 左欄／儲存：……
- 完整例子：……

### §3 成果
- ……

### §4 創新／亮點
- ……

### §5 技術含量
- ……
```

學生回「確認」後，以上條列**原樣**（可微調用字）寫入 `專題報告.md` 對應章節，不得再大幅改寫未告知學生的內容。
